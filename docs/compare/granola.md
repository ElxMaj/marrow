# Marrow vs Granola

Granola turns meetings into searchable notes. Marrow turns meetings into decisions that survive into the codebase.

|  | Granola | Marrow |
|---|---|---|
| Output | Notes + highlights | Decided facts, open questions, contested items, each linked to an evidence span |
| Integration | Notes app, CRM | CLI, MCP server, CI drift checks, watched folders |
| Developer value | Read what was said | Catch code that contradicts what was decided |
| Evidence model | Editable notes | Append-only, immutable evidence with status/confidence/provenance |
| Cost claim | Time saved in note-taking | Tokens saved by task-scoped context; drift caught before it ships |

Use Granola for beautiful meeting notes. Use Marrow when product truth is the constraint your agents build on.
