# Marrow roadmap: July 2027 to July 2028

The year Marrow goes from a beautiful demo to a must-have tool in product engineering
teams. A dependency-ordered todolist executed one CI-green PR at a time. Each box is
one PR (or an explicitly split pair). A box gets ticked in the same PR that ships it,
with the real merged PR number. A box is never ticked ahead of the merge.

## Where this list comes from

A full product test run on 2026-07-17: a seeded brain driven through every console
view in a real browser as both engineer and PM, the landing and live demo walked as a
stranger would, and a 28-agent adversarial audit over six surfaces (CLI, published npm
first-run, HTTP API, static demo, landing, MCP stdio). Twenty-one of twenty-one
serious findings were independently reproduced by verifiers instructed to refute
them; zero were refuted. The full finding set lives in the session record; the five
blocker-class results drive the arcs below.

1. **The repos diverged into two products, and the customer-facing one is the unsafe
   one.** This public repo carries the write-side integrity arc (scrub before insert,
   accept_catch that records without deciding, 17 MCP tools, injection guidance). The
   internal repo that serves the landing, the live demo, and daily dev work has none
   of it: its accept_catch promotes to decided from an agent tool with confidence
   stamped human, and evidence is stored with no secret scrubbing at all.
2. **Keyless retrieval is a dead surface.** With no model key the provider error is
   swallowed and no embedder is wired, so every natural-language query silently
   returns nothing, while the README promises zero-config embeddings. The zero-config
   local embedder exists in the codebase and is never used.
3. **The fixtures pollute real brains.** `demo` and `benchmark` write fictional
   decided facts into whatever DATABASE_URL points at. Evidence is append-only, so
   there is no undo. Keyless `benchmark` then reports a fabricated headline ratio.
4. **The growth surface routes users to a squatter.** Contact mailtos, canonical,
   og:url, sitemap, JSON-LD, and llms.txt all point at a domain owned by a third
   party; og:image 404s; the Cloud waitlist POSTs to a route that does not exist, so
   every signup ever submitted was lost.
5. **Polish gaps a PM catches in an hour.** Every API error is a 500 with internals
   leaked; cross-site POSTs are accepted; the living graph map never reached the
   console visitors see; the static demo's timestamps rot in place; mobile overflows
   at 390px on both public surfaces; the drift CI gate goes green on re-run while the
   violation is still open.

What was confirmed working is the foundation to build on, not rebuild: the promote
beat end to end in the browser, provenance down to the typed answer becoming evidence
mid-flow, landing copy honest to the decimal, CLI error taxonomy with remedies, MCP
reads in 0-18ms, and a static demo that performs the full loop client-side while
saving nothing.

## Sacred rules (every PR is checked against them)

- ONE Postgres plus pgvector. No graph db, Redis, Kafka, queue service, or daemon.
  The graph is an edge table walked by a recursive CTE. Scheduling is cron or GitHub
  Actions calling the CLI.
- Evidence is append-only and immutable. Secrets are scrubbed before the insert
  because there is no editing after it.
- Every distilled fact carries status, confidence, and a verbatim provenance span.
  No fact without a quote.
- Agents propose, humans promote. The only paths to decided are a human answering a
  question or a human authoring a goal. No MCP tool writes any status.
- Context is task-scoped. Nothing returns the whole brain; search stays flat, the
  graph walk lives inside prepare_task only.
- Plain language, no em dashes anywhere the guard scans.

## Execution model (the loop)

1. `git fetch` and rebase onto `origin/main` before branching (history was
   force-pushed once).
2. Branch `feat/<slug>` or `fix/<slug>`, tests first, then code.
3. Definition of done per PR: migrations additive and idempotent; `pnpm db:migrate`,
   `pnpm typecheck`, `pnpm lint`, `pnpm test` (summary must show `ℹ fail 0`
   exactly), `pnpm smoke:packed`, `node scripts/check-benchmark-drift.mjs`, each run
   explicitly, never piped through tail; one changeset when a published package
   changes (the fixed group bumps all five in lockstep); no em dashes; sacred-rule
   proof in the PR body; CI green including drift.
4. Autonomous merge on green for surface, docs, and polish PRs. Pause for human
   review before merge on product-defining PRs: anything touching the decide
   boundary, a migration, retrieval behavior, or a deploy target swap. Founder-gated
   items are marked and never merged by the loop.
5. After a merge: tick the box in the same PR, rebase, take the next item. After each
   arc: redeploy the affected surface and re-verify in a real browser before moving
   on.

### Arc 1. Truth integrity: the write side keeps its promises

- [ ] **R5. Zero-config keyless search** (M, human review: retrieval behavior).
  Wire `LocalEmbeddingProvider` when no model key exists so search is semantic out
  of the box, with a one-time stderr notice about the model download and an env
  opt-out to lexical-only. The README's zero-config claim becomes true. Tests prove
  a keyless brain finds its own just-ingested fact by meaning, not substring.
- [ ] **R6. Fixtures never pollute a real brain** (M, human review: write path).
  `demo` and `benchmark` refuse to write into a brain that already holds real
  evidence unless `--force`; both default to tagging their fixture rows and
  `benchmark` tears its corpus down after measuring. A fresh-database fast path
  keeps the sixty-second demo story intact.
- [x] **R7. Keyless honesty end to end** (S, autonomous). Shipped in PR #91.
  The public benchmark needed no change: it already runs the concept embedding
  in a scratch schema with the committed report's numbers, keyless or keyed.
  `loop` never stamps `safe_to_build` on an empty brief (a brief with zero decided
  facts gates as ask_human_first with an honest reason); keyless empty search states
  name the mode and the remedy; `doctor` surfaces the undistilled backlog count.
- [ ] **R8. No fact without a real quote** (M, human review: store boundary).
  Store-level validation clamps or rejects provenance spans that extend past the
  end of their evidence text, on every insert path including MCP propose_node. A
  lint check flags any existing empty-quote spans. Shared gap, fixed at the one
  choke point.
- [ ] **R9. The drift gate stays red while the violation lives** (M, human review:
  CI semantics).
  `drift --ci` fails while any open catch matches the current diff, not only on
  first sighting; re-running the job cannot launder an unresolved contradiction.
  Fix the GitHub annotation file= prefix so annotations attach to files.

### Arc 2. API and security hygiene

- [x] **R10. A real HTTP error surface** (M, autonomous). Shipped in PR #95.
  Typed 4xx errors with clean messages on every route (400 validation, 404 unknown,
  405 with Allow, 413 too large), one JSON error envelope, internals never leaked,
  HEAD supported. The web console API stops translating every client mistake into a
  500.
- [ ] **R11. The local brain is not CSRF-able** (M, human review: security posture).
  POST routes enforce same-origin (Origin/Host check with an env override for
  intentional LAN serving), content-type is required to be application/json,
  and responses carry Cache-Control, nosniff, frame-deny, and a same-origin CSP.
  A hostile web page can no longer write into a developer's localhost brain.

### Arc 3. The surfaces visitors judge

- [x] **R12. The living map reaches the console people see** (M, autonomous plus
  screenshot review). Shipped in PR #97: the export now bakes the graph into
  the snapshot (it was silently absent, so the demo map claimed an empty
  brain); the served console already defaulted to the map.
  The hand-rolled SVG node-link galaxy (already in this repo's Graph view) ships as
  the default graph experience in the exported demo and the served console, with
  the card grid as the large-graph fallback. Click opens the trace panel. No new
  dependencies.
- [x] **R13. Mobile stops overflowing** (S, autonomous). Shipped in PR #98.
  The landing's 400px column minimum was the real defect; the console was
  already clean at a true 390px viewport (the original finding was partly a
  headless measurement artifact, corrected on the record). The gate probes a
  real 390px iframe instead of screenshots and runs in CI as mobile:check.
  Landing hero CTAs and badge chips stack inside 390px; console question cards fit
  the viewport with Promote reachable.
- [x] **R14. The demo stops aging** (M, autonomous). Shipped in PR #99.
  One deliberate deviation: unknown /api/ paths keep the platform 404 (a
  correct status beats a branded body served as 200; static rewrites cannot
  set both).
  Seeding stamps deterministic relative offsets spread over days (derived from the
  seed date, not one shared second), the demo shows an honest "seeded N days ago"
  line, and a monthly scheduled re-seed plus re-export keeps the hosted snapshot
  fresh. Cache headers on hashed assets become immutable.
- [x] **R15. The console answers "what happens next"** (S, autonomous). Shipped in PR #100.
  Keyless ingest confirmation names the mode and the exact next command (evidence
  stored, model key needed to distill it into facts), and an in-progress catches
  receipt gets a navigation guard (registry checked before nav so cancelling
  preserves the draft) so it is not silently abandoned.

### Arc 5. Must-have for product engineering teams

The strategic arc: the features that turn a polished single-player tool into the
team's shared memory. Sacred rules unchanged: humans promote, one Postgres, agents
propose.

- [ ] **R19. Named humans** (M, human review: decide boundary metadata).
  Every promote records who decided (an identity string from env, flag, or the
  web session), shown in trace and truth output. Teams need to know which human
  stands behind a fact. The promote path itself is unchanged.
- [ ] **R20. The Slack connector closes the loop** (L, human review: connector
  write path).
  One connector shipped end to end and genuinely good: configured channels flow to
  scrubbed, deduped, append-only evidence with permalinks as provenance, synced by
  cron through the CLI, secrets in the existing encrypted store. The room finally
  flows in without copy-paste.
- [ ] **R21. The GitHub connector annotates where teams live** (M, autonomous).
  PR descriptions and review threads become evidence; drift annotations land on
  the pull request with receipts and the paste-ready resolution commands. Marrow
  meets the team inside the tool they already use.
- [ ] **R22. The daily brief becomes the morning read** (M, autonomous).
  `marrow truth` gains an HTML artifact and a Slack-postable digest (via the R20
  connector), so the daily brief of proposed goals, contested facts, gaps, and
  catches arrives where the team starts its day. Read-only, cron-scheduled.
- [ ] **R23. Prove the write side with real numbers** (M, human review: public
  claims). Extend the scorecard with measured write precision from dogfooding:
  duplicate rate, catch precision, dismiss rate, promote latency, all computed
  from the observability tables that already exist and published with methodology.
  The claim no competitor can make, backed the way Marrow backs everything.

### Arc 6. The retest gate

- [ ] **R24. The full retest** (L, autonomous run, human review of the verdict).
  Re-run this audit end to end against the consolidated product: seeded brain,
  full browser walkthrough of every view and flow, mobile sweep, landing and demo
  as a stranger, six-surface adversarial workflow with independent verification.
  Publish the results as `docs/audits/2027-retest.md`. Every finding above must be
  fixed or explicitly waived with a reason. The roadmap is done when the retest
  says outstanding, not when the boxes are ticked.

## Verification

Per PR: the explicit check battery in the execution model. Per arc: redeploy the
affected surface and verify in a real browser, both themes, mobile width included.
The year ends with R24, and R24 ends with the sentence "this is a world-class tool"
being defensible under adversarial review, or the loop continues.
